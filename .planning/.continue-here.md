## Critical Anti-Patterns

- Não interpretar “pronto para deploy futuro” como autorização para produção.
- Não executar deploy, migration remota, seed, reset, restore ou WhatsApp real sem prompt explícito.
- Não imprimir ou versionar `.env`, senha, token, API key, sessão, QR Code, número completo ou `DATABASE_URL`.
- Não usar `git add .` ou `git add -A`; manter commits seletivos.
- Não reutilizar sessão ou segredo local em ambiente futuro.
